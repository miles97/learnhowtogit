###  full stack engineer Development Map

兄弟们，朋友们，现在总结一下前端工程师的技能路线。

是这样的，为了面对需求甚多的组长。我们必须的技能就是HTML5+CSS3,然后根据工作内容的不同，我们的技能水平可以一定的差异，譬如说不需要你学会怎么布局怎么居中，但是需要你bootstrap会用以及会修改成自己部门特定的样式。

CSS完了之后应该是一些简单的过度动画，对于canvas这样高级的要求应该是史前的JS工程师或者H5专业户才会熟练使用的技能，所以对于我们而言。学会JS已经很好惹。

不过我的学习路径是原生JS,大部分人似乎都开始从JQuery开始学习，而我是Angular然后到Vue，最后才开始JS和JQ的学习，现在仅剩的想法就是将VUE学习完毕以及之前的技能树全部点亮之后再开始其他的学习，当然Node是无可避免的需要学习。
关于几个框架的学习水平需要的话，我的确认为JS比较重要，因为万能，然后实际上如果需要代码的话用JQ，不过好多公司已经开始脱离JQ，而且未来的方向都是Weex等基于Vue思想的编程逻辑，组件化可复用的头部尾部，然后不同的内容div id="app"

然后梳理一下付费课程觉得我们需要掌握的技能水平（实际上要说前端的其他框架，感觉深度理解不需要，而且简单上手之后基本上很好使用，所以原生的框架才是真正需要在意的东西）

## 付费课程其一吖
大概就是JS到BOM,DOM,然后简单学习H5，CSS3的一些具体实例，成功加深运用水平，然后
直接使用Node盖房子。最后使用微信小程序模拟简单的全战全栈开发。
然后对于一些细节问题的优化答疑，像是播放器，后台管理，数据交互以及JSON，AJAX的答疑。
通过第一个项目之后获得部分见识认知，然后开始做后台的数据接口，以小见大，了解一个简单的移动app开发流程。
有空再给你普及普及数据交互的深层次概念，不行就教你其他的组件开发，搜索吖，第三方接口吖，爬虫吖，特效吖，最新的技术规范，数据库的使用吖。
这个水平几乎已经成为一个简单的站长，然后没毛病，继续学习bootstrap然后美化优化个人项目。
